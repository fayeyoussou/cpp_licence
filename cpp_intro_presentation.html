Qu'est-ce que le C++ ?

C++ est un langage de programmation créé pour écrire des programmes rapides et puissants.

Caractéristiques principales

  • Langage compilé : votre code est traduit en langage machine avant l'exécution
  • Performance : parmi les langages les plus rapides qui existent
  • Contrôle : vous gérez directement la mémoire et les ressources
  • Polyvalent : peut tout faire, des jeux vidéo aux systèmes d'exploitation

Un Peu d'Histoire

La naissance du C++ (1979-1985)

  • 1979 : Bjarne Stroustrup travaille chez Bell Labs (USA)
  • Il veut améliorer le langage C en ajoutant des classes
  • Nom initial : "C with Classes" (C avec des classes)
  • 1983 : Renommé C++
    • Le "++" signifie "incrémenter" en C → C amélioré !

Évolution

  • 1998 : Premier standard officiel (C++98)
  • 2011 : Grande révolution avec C++11 (C++ moderne)
  • Aujourd'hui : Nouvelles versions tous les 3 ans (C++14, 17, 20, 23...)

Pourquoi le C++ a été créé ?

Le problème avec le C

  • Le langage C est puissant mais difficile pour les gros projets
  • Pas d'organisation du code (pas de classes)
  • Difficile de réutiliser le code
  • Gestion manuelle de tout → erreurs fréquentes

La solution : C++

  • ✅ Garde la vitesse et le contrôle du C
  • ✅ Ajoute l'organisation avec les classes (Programmation Orientée Objet)
  • ✅ Facilite la réutilisation du code (templates)
  • ✅ Reste compatible avec le code C existant

La Philosophie du C++

"C++ vous fait confiance. Il vous donne tous les outils, mais c'est à vous de les utiliser correctement."

Les grands principes

  • Performance d'abord : aucune fonctionnalité ne doit ralentir le programme
  • Liberté totale : vous pouvez faire ce que vous voulez (même des erreurs !)
  • Pas de gaspillage : vous ne payez que pour ce que vous utilisez
  • Expressivité : écrire du code qui ressemble à ce qu'on veut dire

⚠️ Attention : Avec un grand pouvoir vient une grande responsabilité !

Ce que le C++ Peut Faire

Programmation Procédurale (comme le C)

Des fonctions qui s'appellent les unes les autres

Programmation Orientée Objet (POO)

Organiser le code avec des classes et des objets

  • Exemple : une classe Voiture avec des attributs (couleur, vitesse) et des méthodes (démarrer, freiner)

Programmation Générique

Écrire du code qui fonctionne avec plusieurs types de données

  • Exemple : une fonction de tri qui marche pour des nombres, du texte, etc.

Programmation procédurale vs orientée objet

Deux façons complémentaires d’organiser un programme. La première se concentre sur les étapes à exécuter, la seconde sur les entités (objets) qui possèdent des données et des comportements.

Définitions rapides

  • Procédurale : on écrit des fonctions qui manipulent des données passées en paramètres. Idéale pour des scripts, outils, calculs linéaires.
  • POO : on regroupe données + fonctions dans des classes pour former des objets. Adaptée aux systèmes avec plusieurs entités qui interagissent.

Tableau de comparaison

Aspect Procédurale Orientée Objet
Organisation Fonctions + données passées en paramètres Code regroupé en classes et objets
Modélisation Séquence d'étapes (algorithmes) Entités du domaine et leurs interactions
Données Souvent exposées; peu d'encapsulation Encapsulation, invariants protégés
Réutilisation Bibliothèques, fonctions Héritage, composition, polymorphisme
Couplage Peut devenir fort (état global) Interfaces et abstractions pour limiter le couplage
Évolutivité Gère bien les petits/moyens programmes Plus adaptée aux grands projets
Performance Très proche de la machine Peut être aussi rapide si bien conçue
Exemples typiques Scripts systèmes, calculs linéaires Jeux, GUI, systèmes complexes

Quand utiliser quoi ?

  • Procédurale : petites applis linéaires, utilitaires, calculs, scripts, prototypes rapides.
  • POO : projets moyens/grands, interfaces graphiques, jeux, simulateurs, modèles métier riches.
À retenir : la POO facilite la lisibilité et l’évolutivité quand le domaine est riche. Sinon, rester procédural est souvent plus simple et plus clair.

Exemple comparatif (procédural vs POO)

Approche procédurale

#include <iostream>
#include <string>

struct Compte { std::string titulaire; double solde; };

void deposer(Compte& c, double m) { c.solde += m; }
bool retirer(Compte& c, double m) {
    if (m > c.solde) return false; // pas assez
    c.solde -= m; return true;
}

int main() {
    Compte c{"Alice", 100.0};
    deposer(c, 50);
    if (!retirer(c, 200)) std::cout << "Fonds insuffisants\n";
    std::cout << c.titulaire << ": " << c.solde << "\n";
}

Approche orientée objet

#include <iostream>
#include <string>

class Compte {
    std::string titulaire_;
    double solde_{};
public:
    Compte(std::string t, double s) : titulaire_(std::move(t)), solde_(s) {}
    void deposer(double m) { solde_ += m; }
    bool retirer(double m) {
        if (m > solde_) return false; // invariant protégé
        solde_ -= m; return true;
    }
    double solde() const { return solde_; }
    const std::string& titulaire() const { return titulaire_; }
};

int main() {
    Compte c{"Alice", 100.0};
    c.deposer(50);
    if (!c.retirer(200)) std::cout << "Fonds insuffisants\n";
    std::cout << c.titulaire() << ": " << c.solde() << "\n";
}

Ce qu'il faut comprendre

En POO, l'état (données) et les règles (méthodes, invariants) vivent ensemble dans la classe. On manipule l’objet via une interface publique claire, ce qui réduit les erreurs et le couplage.

Pas à pas : procédural

  • Les fonctions deposer et retirer reçoivent la structure Compte et la modifient.
  • Le contrôle (solde suffisant) est dissocié des données et doit être répété partout.
  • Le risque d’erreur augmente si quelqu’un modifie solde sans passer par les bonnes fonctions.

Pas à pas : POO

  • La classe Compte encapsule les données et expose des méthodes (deposer, retirer).
  • Les invariants (ex. ne pas aller sous zéro) sont vérifiés dans la classe.
  • Le code appelant est plus lisible : c.deposer(…) dit ce qu’on fait à l’objet.
Vocabulaire utile :
  • Encapsulation : cacher l’implémentation et protéger les règles internes.
  • Abstraction : ne montrer que ce qui est nécessaire pour utiliser l’objet.
  • Composition : construire des objets à partir d’autres objets (souvent préférable à l’héritage).

Votre Premier Programme C++

#include <iostream>

int main() {
    std::cout << "Bonjour le monde !" << std::endl;
    return 0;
}

Explication ligne par ligne

  • #include <iostream> : inclut la bibliothèque pour afficher du texte
  • int main() : fonction principale, point d'entrée du programme
  • std::cout : affiche du texte à l'écran
  • return 0 : le programme s'est bien terminé

C++ vs C : Quelle Différence ?

Le C++ ajoute au C :

  • Les classes et objets (POO) → meilleure organisation du code
  • Les templates → code réutilisable
  • La STL (Standard Template Library) → plein d'outils prêts (vector, map...)
  • Les exceptions → meilleure gestion des erreurs
  • Les références → alternative plus sûre aux pointeurs
  • Les namespaces → éviter les conflits de noms
Important : C++ peut utiliser du code C, mais pas l'inverse !

Avantages et Inconvénients

✅ Avantages

  • Très rapide : parmi les plus performants
  • Contrôle total : mémoire, ressources
  • Polyvalent : peut tout faire
  • Largement utilisé : dans l'industrie
  • Beaucoup d'outils : bibliothèques disponibles

❌ Inconvénients

  • Complexe : beaucoup de concepts
  • Erreurs difficiles : bugs mémoire
  • Compilation longue : pour gros projets
  • Courbe d'apprentissage : prend du temps
  • Gestion manuelle : plus de travail

De Quoi Avez-vous Besoin ?

Pour programmer en C++

    • GCC, Clang, ou MSVC
